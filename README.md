# 📚 LlamaIndex RAG 论文问答系统

> 专为学术科研场景设计的高阶检索增强生成（RAG）检索引擎

基于 FastAPI + LlamaIndex + 阿里云通义千问大模型构建，创新性引入了层级文档解析、支持动态路由的混合检索架构以及长短期记忆管理，为复杂学术文献的阅读和洞察提供高精度的智能化辅助。

---

## ✨ 核心特性

| 特性 | 说明 |
|------|------|
| 🧠 **层级解析与分块** | 基于 LlamaParse 高精度解析 PDF，利用正则提取 Markdown 标题层级，实现基于父子节点（Parent-Child）的上下文保留分块 |
| 🔍 **高阶混合检索** | 自定义 BM25 关键词 + FAISS 向量语义双路召回，支持 RRF / 权重融合排序，有效规避单一检索的局限性 |
| 🔀 **动态路由与合并** | 独创基于章节标题的路由检索（Routing），并支持父节点自动合并（Auto-merge），大幅提升上下文完整性 |
| ⚡ **智能查询改写** | 内置 QueryProcessor，结合多轮历史对话上下文，将口语化提问自动重写为高精度的学术检索词 |
| 🧠 **长短期记忆管理** | 短期滑动窗口记忆 + 基于 FAISS 的长期向量化记忆，让系统具备强大的长程连贯对话能力 |
| 📊 **RAG 评估流水线** | 内置评估模块，可构建数据集并一键测试 Hit Rate（命中率）和 MRR，方便对比不同检索策略效果 |

##🧩 核心检索机制 (Hybrid Retrieval)
为了应对学术文献的长文本和复杂逻辑，本项目未采用简单的单路向量检索，而是实现了一套精密的管道：

路由检索 (Routing)： 首先在所有提取到的“一级/二级标题”节点中进行检索，定位高度相关的论文章节。

深度召回 (BM25 + Vector)： 锁定章节后，在该章节范围内同时触发基于字词频率的 BM25 检索和基于语义的 FAISS 向量检索。

分数融合与重排 (Rerank)： 采用加权融合（Weighted Fusion）或 RRF 算法合并结果，并调用 gte-rerank 模型进行精准二次重排序。

上下文自愈 (Auto Merge)： 如果检索到大量细碎的子节点（Child Node）均指向同一个父节点（Parent Node），系统会自动将结果替换合并为完整的父节点内容，交由大模型生成答案。

## 📋 系统处理流程
文献上传与高精解析：用户上传 PDF 后，LlamaParse 将其精准转为 Markdown，正则系统提取完整的 H1 > H2 > H3 层级树。

构建层级索引：按照父子分块策略，为短文本计算 Embedding，长文本保留为后备上下文，双路存入本地 FAISS 库与 BM25 词典。

记忆感知与查询改写：接收用户提问，提取长短期记忆中的历史对话，指派大模型将口语提问转化为专业的“学术检索 Query”。

混合召回与生成：执行路由定位 -> 双路召回 -> Rerank 重排 -> 父节点合并，最终将最优质的参考片段拼接，结合 System Prompt 指导大模型以学术口吻输出最终答案，并严格附带引用章节。
